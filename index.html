CSS

    Terms and Definitions

  • Comment // A comment has no effect on the display of the page that's styled, its for the CSS author read and better understand the code. Comments are universal to all programing related languages. example /*this is an old IE6 hack */
  • Style Rule // A style rule (also called a set) is a single section of CSS including the selector, the curly braces, and the different lines with properties and values.
  • Declaration Block // A decloration block is the section of CSS where the property/value pairs appear.
  • Declaration // A declaration is generally any single line of CSS that appears between the curly braces, whether shorthand or longhand.
  • Property // A property is what appears before the colon in any line of CSS.
  • Value // A value is what appears immediately after the colon in any line of CSS.
  • Selector // A selector is the part of the CSS line that selects what element to target with the property/value pair. EX #container #box { width: 200px }
  • Element Type Selector // An element selector is a selector that targets an element by the tag name. The selector in the example below is an element type selector, because it doesn't use a class, ID, or other selector to apply the given styles. nav {}
  • Class Selector // The selector in the example targets an element by its class name. EX /* matches HTML element with class="navigation" */ .navigation {}
  • ID Selector // The selector in the example targets an element by its ID. So every element with an ID of "navigation" will receive the styles in question. Ex /* matches HTML element with id="navigation" #navigation {}
  • Universal Selector // The universal selector matches any element within the context in which it's placed in a selector. In the example, the * character is the universal selector. EX /* the asterisk character is the universal selector */ .navigatoin ul * {} So any elements that appear as a child descendant of the unorderlist element insande an element that has a class of "navigation" will receive the declared styles. Universal selectors are generally discouraged for performance reasons.
  • Attribute Selector // An attribute selector selcts an element to style based on an attribute and/or attribute value. EX /* matches element with style ="[anything]" */ p[style] // 2nd EX /* matches input elements with type ="text" */ input[type="text"]
  • Pseudo-Class // A pseudo-class works similary to a regular CSS class, except it's not explicitily declared in the HTML. EX /* the word "hover" along with the preceding colon is the PC */ a:hover / A PC always has a single colon followed by a keyword of some sort, with no space before or after the colon
  • Pseudo-Element // A PE is not exactly the same as a PC. While a PC matches elements that actually exist, PE target "virtual" elements that can change depending on the actual HTML. CSS2 PE use a single colon, and CSS3 PE use a double colon
  • Cominator // A combinator is the character in a selector that connects two selectors together. There are FOUR types of combinators. These four combinators help create descendant selectors (with a space character), child selectors (with the ">"character, adjacent sibling selectors (with the "+" character), and general sibling selectors (with the "~"character). EX div p / div>p / div+p / div~p
  • At-Rule // An at-rule is an instruction given in a CDD document using the @ character. An at-rule could have a declaration block or a simple string of text. EX @import url(secondary.css) @media print {}
  • Statement // A statement in CSS is any at-rule or rule set.
  • Identifier // An identifier can be anything that appears as a property, id, class, keyword value, and at-rule EX /* "body" , "background", "none", and "font-size" are identifiers
  • Keyword // A keyword is a value for a property and is somewhat like a reserved word for a particular property. Different properties have different keywords, and all properties allow the keyworkd "inherit" EX #container { height: auto; /* "auto" is a keyword */}

Git

    SETUP

    Configuring user information used across all local repositories

  • git config --global user.name "[firstname lastname]" // Sets a name that is identifiable for credit when review version history
  • git config --global user.email "[valid-email]" // Set an email address that will be associated with each history marker
  • git config --global color.ui auto // set automatic command line coloring for Git for easy reviewing

    • SETUP and INIT

  • git init // initialize an existing directory as a Git repository
  • git clone [url] // retrieve an entire repository from a hosted location via URL

    • STAGE AND SNAPSHOT

    Configure user information used across all local repositories

  • git status // shows modified files in working directory, staged for your next commit
  • git add [file] // adds a file as it looks now to your commit (stage)
  • git reset [file] //
  • git diff // diff of what is changed but not staged
  • git diff --staged // diff of what is staged but not yet committed
  • git commit -m "[descriptive message]" // commit your staged content as a new commit snapshot

    • BRANCH AND MERGE

    Isolating work in branches, changing context, and intergrating changes

  • git branch // lists your branch a* will appear next to the currently active branch
  • git branch [branch-name] // create a new branch at the current commit
  • git checkout // swithch to another branch and check it out into your working directory
  • git checkout -b branch name: creates a new branch and switches to it
  • git merge [branch] // merge the specified branch's history into the current one
  • git log // shows all commits in the current branch's history

    • INSPECT AND COMPARE

    Examining logs, diffs and object information

  • git log // show the commit history for the currently active branch
  • git log branchB..branchA // show the commits on branchA that are not on branchB
  • git log --follow [file] // show the commits that changed file, even across renames
  • git diff branchB..branchA // show the diff of what is in branchA that is not in branchB
  • git show [SHA] // show any object in Git in human-readable format

    • TRACKING PATH CHANGES

    Versioning file removes and path changes

  • git rm [file] // delete the file from project and stage the removal for commit
  • git mv [existing-path] [new-path] // change an existing path and stage the move
  • git log --stat -M // show all commit logs with indication of any paths that moved

    • IGNORING PATTERNS

    Preventing unintentional staging or commiting of files

  • logs/*.notespattern*/ // save a file with desired patterns as .gigignore with either direct string matches or wildcard globs
  • git config --global core.excludefile [file] // system wide ignore pattern for all local repositories

    • SHARE AND UPDATE

    Retrieving updates from another repository and updating local repos

  • git remote add [alias] [url] // add a git URL as an alias
  • git fetch [alias] // fetch down all the branches from that Git remote
  • git merge [alias]/[branch] // merge a remote branch into your current branch and bring it up to date
  • git push [alias] [branch] // Transmit local branch comnmits to the remote repositry branch
  • git pull // fetch and merge any commits from the tracking remote branch

    • REWRITE HISTORY

    Rewriting branches, updating commits and clearing history

  • git rebase [branch] // apply any commits of current branch ahead of specified one
  • git reset --hard [commit] // clear staging area, rewrite working tree from specified commit

    • TEMPORARY COMMITS

    Temporarily store modified, tracked files in order to change branches

  • git stash // Save modified and staged changes
  • git stash list // list stack-order of stashed file changes
  • git stash pop // write working from top of stash stack
  • git stash drop // discard the changes from top of stash stack

JavaScript

    JavaScript Variables

  • var: The most used variable. Can be reassigned but only accessed within a function, meaning function scope. Variable defined with var move to the top when code is executed.
  • const: Cannot be reassigned and not accessible before they appear within the code, meaning block scope.
  • let: simliar to const with block scope, however the let variable can be reassigned but not re-declared.

    • Date Types

  • Numbers:var age = 33
  • Variables: var a
  • Text (strings): var a = "Sachin"
  • Operations: var b = 4 + 5 + 6
  • True or False Statements: var a = true
  • Constant Numbers: const PI - 3.14
  • Objects: var fullName = {firstName: "Sachin", lastName: "Samal"}

    • Objects

  • This is a simple example of objects in JS. This object describe the variable car, and includes KEYS or PROPERTIES such as make, model, and year. Each property has a value such as Nissan, Altima, and 2022. A JS object is a collection of properties with values, and it functions as a method.

  • break // Stop and exit the cycle at certain conditions

    • if-else statements

    An if statement exectues the code within brackets as long as the condition in parenthesis is true. Failing that, an optional else statement is exectued instead.

    Example if (condition) { // do this if condition is met } else { // do this if condition is not met }
